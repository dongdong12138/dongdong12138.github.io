提升打包构建速度

HotModuleReplacement（HMR）​

开发时，我们修改了其中一个模块代码，Webpack 默认会将所有模块全部重新打包编译，速度很慢。

所以我们希望修改某个模块代码，就只有这个模块代码需要重新打包编译，其他模块不变，这样就可以提升打包速度。

HotModuleReplacement（HMR/热模块替换）：在程序运行中，替换、添加或删除模块，而无需重新加载整个页面。

基本配置
module.exports = {
   // ...
   devServer: {
      host: 'localhost',
      port: '3000',
      open: true,
      hot: true,    // 开启 HMR 功能（只能用于开发环境，生产环境不需要），默认就是 true
   },
}

此时 css 样式经过 style-loader（已经具备 HMR 功能）处理，已经具备 HMR 功能了，但是 js 还不行。

JS 配置（main.js）
import count from './js/count'
import sum from './js/sum'

const result1 = count(2, 1)
console.log(result1)
const result2 = sum(1, 2, 3, 4)
console.log(result2)

// 判断浏览器是否支持 HMR 功能（以下代码生产模式下会删除）
if (module.hot) {
   // 当下面两个 js 模块发生变化，浏览器会自动更新对应的模块
   module.hot.accept('./js/count.js')
   module.hot.accept('./js/sum.js')
}

上面这样写会很麻烦，所以实际开发中，我们会使用 loader 来解决，比如：Vue Loader、React Hot Loader。

OneOf​

打包时，Webpack 会将每个文件与所有 loader 都匹配一遍，虽然因为 test 正则原因不是所有 loader 都能匹配上，但是都要过一遍，效率比较低。

OneOf 可以指定只能匹配一个 loader，剩下的就不匹配了。

module.exports = {
  // ...
  module: {
    rules: [
      {
        // 文件只被 oneOf 数组中的一个 loader 处理
        oneOf: [
          {
            test: /\.css$/,
            use: ['style-loader', 'css-loader'],
          },
          {
            test: /\.less$/,
            use: ['style-loader', 'css-loader', 'less-loader'],
          },
          {
            test: /\.s[ac]ss$/,
            use: ['style-loader', 'css-loader', 'sass-loader'],
          },
          {
            test: /\.styl$/,
            use: ['style-loader', 'css-loader', 'stylus-loader'],
          },
        ],
      },
    ],
  },
  // ...
}

Include/Exclude​

开发时，我们使用的第三方的库或插件都下载到 node_modules 中了，这些文件已经编译过了可以直接使用，所以我们在对 js 文件处理时，要排除 node_modules 中的文件。

  • include（包含）：只处理 xxx 文件
  • exclude（排除）：除了 xxx 文件以外其他文件都处理
const path = require('path')
const ESLintWebpackPlugin = require('eslint-webpack-plugin')

module.exports = {
  // ...
  module: {
    rules: [
      {
        oneOf: [
          {
            test: /\.js$/,
            // exclude: /node_modules/,   // 排除 node_modules 代码不编译
            include: path.resolve(__dirname, '../src'),   // 也可以用包含，只编译 src 目录中的文件
            loader: 'babel-loader',
          },
        ],
      },
    ],
  },
  plugins: [
    new ESLintWebpackPlugin({
      context: path.resolve(__dirname, '../src'),   // 指定检查文件的根目录
      exclude: 'node_modules',    // 默认值，排除 node_modules 目录不检查
    }),
  ],
  // ...
}

Cache​

每次打包时 js 文件都要经过 ESLint 检查 和 Babel 编译，速度比较慢。我们可以缓存之前的 ESLint 检查和 Babel 编译结果，对于修改的文件才去做 ESLint 检查和 Babel 编译，其他没有修改的文件使用之前的缓存结果，以此提升打包速度。

const path = require('path')
const ESLintWebpackPlugin = require('eslint-webpack-plugin')

module.exports = {
  // ...
  module: {
    rules: [
      {
        oneOf: [
          {
            test: /\.js$/,
            include: path.resolve(__dirname, '../src'),
            loader: 'babel-loader',
            options: {
              // 开启 babel 编译缓存
              cacheDirectory: true,
              // 缓存文件不需要压缩，因为生产环境用不上这些缓存文件，而且压缩缓存文件也是需要消耗时间的
              cacheCompression: false,
            },
          },
        ],
      },
    ],
  },
  plugins: [
    new ESLintWebpackPlugin({
      // 指定检查文件的根目录
      context: path.resolve(__dirname, '../src'),
      exclude: 'node_modules',
      // 开启缓存
      cache: true,
      // 缓存目录
      cacheLocation: path.resolve(__dirname, '../node_modules/.cache/.eslintcache'),
    }),
  ],
   // ...
}

Thread​

当项目越来越庞大时，打包速度越来越慢，甚至需要一个下午才能打包出来代码。

我们想要继续提升打包速度，其实就是要提升 js 的打包速度，因为其他文件都比较少。而对 js 文件的处理主要就是 ESLint、Babel、Terser 三个工具，所以我们要提升它们的运行速度。

备注

terser-webpack-plugin 是 Webpack 内置的插件，用于对 JS 代码进行压缩和混淆。

  • 代码压缩：它可以通过删除空格、注释和其他不必要的字符，以及简化代码来减小 JavaScript 文件的大小，这有助于提高网页加载速度
  • 代码混淆：它可以将变量名、函数名和属性名等重命名为更短、难以理解的名称，从而使源代码难以被理解和逆向工程，这有助于保护你的代码知识产权和安全性
  • 删除未使用的代码：它分析代码中的依赖关系，只保留被使用到的部分，从而删除未使用的代码，减少文件体积

我们可以开启多进程同时处理 js 文件，这样速度就比之前的单进程打包更快了。

所谓多进程打包，就是开启电脑的多个进程（多个 CPU 核数）同时干一件事，速度更快。

注意
  • 请仅在特别耗时的操作中使用，因为每个进程启动就有大约 600ms 左右的开销
  • 建议只针对大文件进行多进程打包，如果文件体积比较小，就不值得使用多进程，否则可能会浪费更多时间

  1. 安装 thread-loader

    npm i thread-loader -D

  2. 使用

    const os = require('os')   // nodejs 核心模块，直接使用
    const path = require('path')
    const ESLintWebpackPlugin = require('eslint-webpack-plugin')
    const CssMinimizerPlugin = require('css-minimizer-webpack-plugin')
    const TerserPlugin = require('terser-webpack-plugin')

    // 获取 cpu 核数
    const threads = os.cpus().length

    module.exports = {
      // ...
      module: {
        rules: [
          {
            oneOf: [
              {
                test: /\.js$/,
                include: path.resolve(__dirname, '../src'),
                use: [
                  {
                    loader: 'thread-loader',    // 开启多进程
                    options: {
                      workers: threads,         // 进程数量
                    },
                  },
                  {
                    loader: 'babel-loader',
                    options: {
                      cacheDirectory: true,   // 开启 babel 编译缓存
                    },
                  },
                ],
              },
            ],
          },
        ],
      },
      plugins: [
        new ESLintWebpackPlugin({
          context: path.resolve(__dirname, '../src'),   // 指定检查文件的根目录
          exclude: 'node_modules',    // 默认值
          cache: true,    // 开启缓存
          cacheLocation: path.resolve(__dirname, '../node_modules/.cache/.eslintcache'),    // 缓存目录
          threads,    // 开启多进程
        }),
      ],
      optimization: {
        minimize: true,
        minimizer: [
          // css 压缩也可以写到 optimization.minimizer 里面
          new CssMinimizerPlugin(),
          // 生产模式会默认开启 TerserPlugin，但是我们需要进行其他配置，就要重新写了
          new TerserPlugin({
            parallel: threads   // 开启多进程
          })
        ],
      },
    }
